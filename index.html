Core product

Six stages. One IFC model.

A strictly sequential pipeline where each stage has one input and one output contract — making every component independently testable, replaceable, and improvable.

01
Universal plan ingestion
Accepts PDF, JPEG/PNG, DWG, and scanned hand-drawn plans. Auto-deskews, detects scale bars, normalizes orientation, and segments multi-page documents — no user intervention required.

Semantic element detection
CNN-based wall and opening detector classifies elements with type metadata — load-bearing vs partition, door swing, window sill. Builds a scene graph of rooms and adjacency, not just raw shapes.

Height + story inference
Derives full 3D geometry from 2D data automatically using topology analysis and residential height tables. Detects stairs, stacks multi-story plans, generates wall extrusions with per-element confidence scores.

Construction-grade IFC export
Maps geometry to ISO 16739 IFC 4 schema with proper classification — IfcWallStandardCase, IfcSlab, relationship sets. Output drops cleanly into Revit or ArchiCAD without reclassification. Also exports RVT, DXF, OBJ, and QTO CSV.

Lead with confidence, hide the magic

Summary · cross-cutting recommendations

Six rules that hold across every stage

If the team only takes six things from these mockups, take these.

01 / Trust
Show confidence, not magic
Every auto-detected value gets a visible confidence indicator. No black-box "we got it right".
02 / Flow
Persistent stepper across all 6
Same six-step bar at the top of every screen. The architect always knows where they are.
03 / Triage
Surface the 10% that needs attention
Lists, not heatmaps. Click-to-jump, not pan-and-scan. Architects review by exception.
04 / Schema
Speak IFC, not "3D file"
IfcWallStandardCase, IfcSpace, relationship sets visible in S4. Pros notice.
05 / Flywheel
Make the loop visible
Show that user corrections feed retraining. Turn a UX moment into a brand moment.
06 / Exit
Finalize is one click from anywhere
Architects should be able to ship a "good enough" model in 60 seconds. Don't gate on perfection.

Risks + mitigation

Known risks, concrete mitigations

Every risk has a playbook. None are dealbreakers if addressed proactively.

RiskTypeMitigation
AI accuracy falls short on noisy plansTechnicalHuman-in-loop editor + iterative fine-tuning. Start with clean single-story plans, expand complexity over time.
Insufficient annotated training dataDataLeverage public datasets (Semantic FloorPlan), crowdsource labeling, partner with firms for real CAD plans + BIM pairs.
Incumbents or funded startups emergeMarketBuild IFC compliance moat + correction flywheel early. Focus residential segments big players ignore.
AEC industry conservative, low adoptionAdoptionEmphasize transparency (show model, allow edits). Target early adopters. Pilot success stories build credibility.
IP / privacy concerns on uploaded plansLegalClear ToS: user owns uploaded plans. Data secured + deleted on request. GDPR-compliant hosting.
IFC schema changes or version fragmentationTechnicalMaintain IFC 2x3 + IFC 4 support. Track buildingSMART releases. Abstract schema layer in codebase.
